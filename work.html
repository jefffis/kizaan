  • PRODUCT DEVELOPMENT
    Acquia Lift is a branded as tool for marketers to use without much technical or development help from you team.

    Actual Experience:

    1. Acquia Lift is not very user-friendly: especially when first trying to create campaigns. It will take multiple training sessions for casual users to understand and be ready to use the tool. Having developer resources is a must during initial implementation and set-up.
      Recommendation: Create a 15/30 day implementation plan. Provide good/educational documentation (best practices, In-App Help / Guidance tutorials) using a project management app (like Basecamp) during the implementation phase to support Lift setup and on-boarding.
    2. Profile development: (such as segmenting member vs non-member data) contains too much back-end involvement. Knowing how to write code should not be a requirement in the campaign building process. Implementing simple variable functionality should be done with a click of a button, not by writing code manually.
      Recommendation: More training options or refresher webinars. Maybe offering a easy-to-see resource area on the dashboard.
    3. Debugging: when setting up within Drupal environment, error messages and the information given does not provide enough information to troubleshoot without seeking technical help from dev team.
      Recommendation: Better error messages with more information. Better training on error messages that might occur. A FAQ on error messages that happen and how to solve them.
    4. Organization: with so many forms, lists, segments and campaigns, it can quickly become a mess of trying to keep track of what has been created and how to manage all the marketing assets.
      Recommendation: Create folder options to quickly organize marketing assets with easy to add folder option in Lift module.
    5. Landing Pages: Building landing pages and adding campaigns isn't as easy as implied during demo. Navigating Lift module can be difficult, as well as knowing exactly what some functions actually do requires constant research on other pages.
      Recommendation: Offering a easy to see resource area on the dashboard being used and descriptions on specific feature and how they relate to other functions.
